Crediclass Dashboard — Documentação Técnica

Sistema de análise de grupos de consórcio e geração de Estudo Financeiro — versão 1.0 — 2025/2026

Este painel é uma ferramenta interna da Crediclass para consultores de crédito. O objetivo é cruzar o perfil financeiro do cliente (extraído automaticamente do CRM Piperun) com os dados dos grupos de consórcio disponíveis (extraídos de planilha Google Sheets) e identificar os grupos mais compatíveis para compor o Estudo Financeiro.

Stack: Python 3.12 + FastAPI — Alpine.js 3.x — Tailwind CSS (CDN) — Chart.js 4.4.3 — Google Sheets API v4 — Piperun REST API v1

Lógica de cache

A função fetch_grupos(force_refresh=False) verifica se o cache existe e tem menos de 1 hora. Se sim, retorna o JSON do disco. Caso contrário (ou se force_refresh=True), busca a planilha, processa e grava o novo cache.

Headers: { "token": "...", "accept": "application/json" }

Lógica de extração

Entre todas as notas da oportunidade, o sistema localiza a que contém "DADOS DO FORMULÁRIO" no texto. A função parse_note_text(html_text) processa essa nota:

  1. Remove tags HTML (<br> → newline, remove demais tags)
  2. Divide por linha e busca o padrão Label: Valor
  3. Mapeia labels via FIELD_MAP (≥25 campos)
  4. Converte campos numéricos (formato brasileiro: 600.000,00 → 600000.0)
  5. Calcula pct_lance_disponivel = lance_maximo / valor_imovel × 100

Regras de Negócio — Perfis de Lance

Cada grupo tem 5 indicadores de lance mínimo, correspondendo ao perfil de urgência do cliente. Os nomes das colunas na planilha são mapeados para esses 5 perfis:

🔴 Agressivo
Urgência: 1–3 meses
Lance médio: ≥ 50%
Campo: agressivo_3m
🟠 Moderado
Urgência: 4–6 meses
Lance médio: 40–50%
Campo: moderado_12m
🟡 Conservador
Urgência: 7–12 meses
Lance médio: 30–40%
Campo: conservador_24m
🔵 Super Conservador
Urgência: 13–24 meses
Lance médio: 20–30%
Campo: não exibido separadamente
🟢 Investidor
Urgência: sem pressa
Lance médio: 0–20%
Campo: investidor
No modal de detalhe do grupo, cada perfil exibe ✓ (compatível) ou ✗ (incompatível) com base no pct_lance_disponivel do cliente.

Regras de Negócio — Score de Compatibilidade

A função compatibilidade(g) retorna um score de 0 a 3 pontos, avaliando três critérios independentes:

CritérioCondiçãoPontos
Cobertura de crédito grupo.maior_credito ≥ valor_imovel × 0.9 +1
Orçamento mensal grupo.parcela_inicial ≤ mensalidade_maxima × 1.1 +1
Lance disponível pct_lance_disponivel ≥ melhor_perfil_lance_do_grupo +1

Lógica do critério de lance

O "melhor perfil" é o menor lance exigido entre os perfis disponíveis no grupo, na ordem: investidor → conservador_24m → moderado_12m. Ou seja, se o grupo tem uma opção com lance baixo (Investidor), e o cliente tem esse lance disponível, o critério é satisfeito.

Filtro inteligente (filtrarCompativeis)

O toggle "Mostrar apenas compatíveis" aplica dois filtros simultâneos na lista de grupos:

  • Crédito cobre pelo menos 90% do valor do imóvel
  • Parcela inicial não ultrapassa 110% da mensalidade máxima declarada

Este filtro é ativado automaticamente ao buscar uma oportunidade Piperun que contenha o campo valor_imovel_num.

Regras de Negócio — Fórmulas (f)(g)(h)

Fórmulas utilizadas na análise de viabilidade e no Estudo Financeiro. Aplicadas por grupo + perfil de lance.

(f) Crédito a contratar crédito = valor_imovel / (1 - pct_lance_embutido)

Onde pct_lance_embutido varia por ADM (ver tabela Lance Embutido).
Objetivo: dimensionar o crédito real necessário considerando que parte do lance sai do próprio crédito.
(g) Lance máximo % pct_lance_maximo = (lance_recursos_proprios + lance_embutido) / credito × 100

lance_embutido = credito × pct_embutido_adm
pct_lance_maximo indica qual perfil de lance é acessível para o cliente neste grupo.
(h) Prazo mínimo de contemplação Com base em pct_lance_maximo, consultar na coluna do grupo qual perfil é atingido:
≥ agressivo_3m → contemplação em 1-3 meses
≥ moderado_12m → contemplação em 4-6 meses
≥ conservador_24m → contemplação em 7-12 meses
≥ investidor → contemplação em 13-24 meses
< investidor → apenas sorteio (prazo indefinido)

Regras de Negócio — Regra da Parcela (30% da Renda)

A parcela do consórcio não deve ultrapassar 30% da renda mensal familiar do cliente.

Parcela máxima permitida parcela_maxima = renda_mensal_num × 0.30

No dashboard, quando a parcela inicial do grupo supera 30% da renda declarada pelo cliente:

  • A célula da parcela é exibida em vermelho no modal de detalhe
  • O critério de orçamento mensal usa mensalidade_maxima (campo do formulário), não a regra dos 30% — pois o cliente pode ter declarado um orçamento diferente
Versão futura: alertar visualmente na tabela principal quando a parcela ultrapassa 30% da renda.

Regras de Negócio — Lance Embutido

Instalação e Execução

Requisitos

  • Python 3.10+
  • Arquivo credentials.json do Google Cloud Console (OAuth 2.0) em backend/

Setup inicial

cd C:\Users\User\crediclass-dashboard-grupos\backend

# Criar e ativar ambiente virtual
python -m venv venv
venv\Scripts\activate

# Instalar dependências
pip install -r requirements.txt

# Iniciar servidor
python main.py
# Acesse: http://localhost:8000

Primeira execução (Google OAuth)

Na primeira vez, o sistema abrirá o navegador para autenticação Google. Depois de autorizar, o token.json é gerado e reutilizado automaticamente.

Alternativa sem OAuth: Se não houver credentials.json, popular o arquivo data/grupos.json manualmente (via script Python com openpyxl + XLSX da planilha). O sistema usa o cache e não precisa da API Google.

Atualizar Dados dos Grupos

Via painel (automático)

Clique no botão Atualizar no header do dashboard. O endpoint POST /api/refresh força a recarga da planilha, invalida o cache e grava o novo grupos.json.

Via planilha XLSX (manual)

Se a API Google não estiver configurada, fazer o download da planilha como XLSX e processar com o script:

cd backend
python processar_xlsx.py  # (se disponível)
# Ou reimportar via MCP Google Drive
