⏳ O QUE VOCÊ PRECISA FAZER (MANUAL):
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

FASE 1 — CLOUDFLARE (5 min)
  → DNS → Records → + Add Record
  → Type: CNAME | Name: crediclass | Content: cname.vercel.app
  → Proxy: Proxied (laranja) | TTL: Auto
  → Save

FASE 2 — VERCEL (10 min)
  → Add New → Project → Import GitHub
  → Select: crediclass-dashboard-grupos
  → Root Directory: frontend/
  → Deploy
  → Settings → Domains → Add: crediclass.csrtecnologia.com.br
  → Aguarde validação (5-10 min)

FASE 3 — RAILWAY/HEROKU (15 min)
  Option A (Recomendado): railway.app
    → New Project → Deploy from GitHub
    → Root: backend/
    → Variáveis ENV (veja backend/.env.example)
    → Deploy

  Option B: heroku.com
    → Create New App
    → Connect GitHub
    → Config Vars (veja backend/.env.example)
    → Deploy Branch

FASE 4 — INTEGRAÇÃO (5 min)
  → Após obter URL do backend (ex: seu-app.railway.app)
  → Editar vercel.json → substituir "your-backend-railway.app"
  → Commit e push
  → Vercel faz redeploy automático
